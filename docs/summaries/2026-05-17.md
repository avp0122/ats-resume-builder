SEO refinement day. Three small PRs + a manual conflict resolution. No code-quality changes — purely surface area for search + answer engines.

## Completed

- **`<title>` shortened to ≤60 chars** (PR #34, merged). Old: `kairesume — The cheapest AI resume builder — free ATS-tailored resumes & cover letters` (85 chars, Google SERP was truncating the second half). New: `kairesume — Free AI ATS resume & cover letter builder` (54 chars, fits cleanly). One-line `SITE_TAGLINE` change in `app/layout.tsx` — feeds `<title>`, `og:title`, `twitter:title` via existing wiring.

- **Explicit AI crawler allowlist in robots.txt** (PR #35, merged). The wildcard `User-agent: *` already covered every UA, but AEO/GEO best practice is to list AI crawlers explicitly — many of them look for their own UA first before falling back to the wildcard. Listed: OpenAI (GPTBot, ChatGPT-User, OAI-SearchBot), Anthropic (ClaudeBot, anthropic-ai, Claude-Web), Common Crawl (CCBot), Perplexity, Google-Extended (Gemini training opt-in token), Applebot-Extended, Meta-ExternalAgent, Bytespider, Amazonbot, DuckAssistBot, YouBot, cohere-ai, mistral-ai. 14 explicit blocks; wildcard stays at the top of `rules[]`.

- **Person JSON-LD for the founder** (PR #36, merged). Fifth structured-data blob in `<head>` alongside Organization / WebApplication / HowTo / FAQPage. Answer engines (Perplexity, ChatGPT, Gemini, Claude) use Person schema to link the product to a real human — useful for trust signals and "who built kairesume?" type queries. Editable `FOUNDER` constant in `app/layout.tsx` (name, jobTitle, knowsAbout, sameAs URLs) so the identity is one block to update. Empty `sameAs` URLs auto-filtered (no broken-link entries).

Workaround: `gh auth switch -h github.com -u avp0122` before each PR-create call. Should add this to the maintenance docs.
