## Important changes

- **Documentation is now a first-class deliverable.** 9 new DECISIONS landed today (021–029). Every non-trivial PR adds one. Append-only — reversed decisions stay with a "Supersedes" entry.
- **Stacked PR merge order matters.** GitHub merges the PR's head into its declared *base branch*, not master. If the base is a sibling branch (typical for stacked review), the content doesn't promote until the sibling itself merges to master. Either (a) merge bottom-up and delete branches as you go so GitHub auto-retargets, or (b) explicitly retarget each PR's base to master before merging.
- **Production schema can have CHECK constraints not present in `supabase/migrations/`.** The `profiles_plan_check` constraint was added at table creation, before this repo's tracked migrations. `\d+ public.profiles` in psql or the Supabase Table Editor's constraint inspector both surface them.
- **Cover letter quality has a floor: humanized JD-only.** Tavily enrichment adds a ceiling. Every cover letter generated ships at least at the floor (DECISION 027) — the enrichment layer (DECISION 028) is purely additive and fails soft to the floor.
- **Cross-OS git interop quirks.** Windows-side Git Bash and WSL's git disagree on CRLF/LF for files normalized one way and edited the other. `git checkout -- .` from the "incorrect" side resets the worktree to the index's view. Worth knowing for any future cross-environment work.
