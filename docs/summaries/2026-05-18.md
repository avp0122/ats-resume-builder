The behavioural shift ships with that line — all gates already read the constant. The bulk of this PR was a sweep of hard-coded "3" mentions across user-facing copy that didn't use the constant:

- **DOCX output alongside PDF** (PR #38, merged). ZIP now contains 4 files: `<name>_resume.pdf`, `<name>_resume.docx`, `<name>_coverletter.pdf`, `<name>_coverletter.docx`. DOCX is what most ATS parsers (Workday, Greenhouse, Lever, iCIMS, Taleo) actually prefer — they read it as real Office Open XML rather than re-deriving text from PDF glyph positions. New dep `@turbodocx/html-to-docx` (browser-safe, modern OOXML, ~80 KB gzipped, dynamically imported only when Download is clicked — zero hit on initial page load). New `renderDocxBlob()` mirrors `renderPdfBlob()`. All four blobs render in parallel inside `Promise.all` — total prep time is max-of-four, not sum.

## Important changes

- **All quota / pricing constants are now the single source of truth.** `lib/pricing.ts` is the only place to change numbers; everything from API errors to JSON-LD `Offer.description` to FAQ copy interpolates from it.
- **Recruiters can now choose their preferred format.** PDF for canonical print/email; DOCX for ATS submission flow. Both ship every time.
