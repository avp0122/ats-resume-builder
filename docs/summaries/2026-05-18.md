# Session summary — 2026-05-18

## Completed

### Shipped to production (merged)
- **Quota banner refresh after each generate** (PR #31) — the home page's `RemainingBanner` / `UpgradePromo` was fetching quota once on mount and never refreshing. After hitting the 3-of-3 cap a user would see two contradictory messages at once: "2 of 3 remaining" mid-page and "You've used all 3 free generations" under the form. Fix: new `applyQuotaFromResponse()` helper maps every `/api/generate` response's `usage` block into `QuotaInfo` and calls `setQuota()` — runs on both the success path and the 402 path.
- **Prompt: PRESERVE COMPLETENESS rule** (PR #32) — Llama 3.3 was over-summarising. User uploaded an 8-job / 11-skill-category resume and got back 2 jobs with 1 bullet each + 5 generic skill labels. Token budget wasn't the issue (output budget 6K, model used ~200). Root cause: prompt said "REWRITE for ATS" with no completeness constraint. Added explicit rules: every job appears, ≥ same bullet count per job, every skill category preserved, ≥90% of original length. Self-check at the bottom: "if your output is materially shorter, you have summarised — restore."
- **PDF page-break: no mid-glyph slicing** (PR #33) — Education entries were getting horizontally sliced through the glyphs across page breaks. Cause: an earlier round removed `page-break-inside: avoid` from `<p>` (to fix a blank-page-1 bug that was actually caused by `'avoid-all'` pagebreak mode cascading the rule onto wrappers). With `'avoid-all'` gone, `<p>`-only avoid is safe — added back. Same on `<h3>` so wrapped job titles don't split.
- **`<title>` shortened to ≤60 chars** (PR #34) — old title was 85 chars and getting truncated in Google SERPs. New: `kairesume — Free AI ATS resume & cover letter builder` (54 chars). Single constant `SITE_TAGLINE` flows into `<title>`, `og:title`, `twitter:title`.
- **Explicit AI crawler allowlist** (PR #35) — 14 named UAs in `robots.txt` (GPTBot, ClaudeBot, CCBot, PerplexityBot, Google-Extended, Applebot-Extended, etc.). Wildcard `*` stays as fallback. AEO/GEO best practice — many AI crawlers look for their own UA before fallback.
- **Person JSON-LD for founder** (PR #36) — fifth schema in `<head>` (Organization, WebApplication, HowTo, FAQPage, Person). Editable `FOUNDER` constant in `app/layout.tsx` so identity is one block to update.
