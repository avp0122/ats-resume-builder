# Session summary — 2026-05-16

Three PRs. Quota UX fix, prompt completeness fix, PDF page-break refinement.

## Completed

- **Quota banner refresh after each generate** (PR #31, merged). Bug: a signed-in free user who just hit the 3-of-3 cap saw two contradictory messages at once — mid-page banner "2 of 3 free generations remaining" AND error under the form "You've used all 3 free generations". Root cause: `quota` state was fetched once on mount from `/api/usage` and never refreshed; the server-side counter incremented but the client banner stayed stale. Fix: new `applyQuotaFromResponse()` helper that maps every `/api/generate` response's `usage` block (success OR 402) into `QuotaInfo` and calls `setQuota()`. Banner now flips from `RemainingBanner` to `UpgradePromo` instantly when `remaining === 0`.

- **Prompt: PRESERVE COMPLETENESS rule** (PR #32, merged). User uploaded a real 3-page resume (8 jobs, 11 skill categories, 40+ bullets) and got back 5 generic skill labels, 2 jobs (1 bullet each), 1 education entry. Not a token-budget issue — input was ~3.4K, output budget 6K, model used ~200 tokens. The prompt told the model to "rewrite for ATS" with no instruction to preserve completeness; Llama 3.3 interpreted that as "summarise." Added hard rules to `lib/prompts.ts`:
  - Every job from the original must appear, same order.
  - Each job gets at least as many bullets as the original.
  - Every skill category preserved with specific technologies (no collapsing into generic labels like "Cloud Computing").
  - Every education / certification / language / award preserved.
  - Length check: rewritten resume must be within ~10% of original.
  - Self-check: "If your output is materially shorter, you have summarised — restore."

- **PDF page-break: no mid-glyph slicing** (PR #33, merged). Education line "MS Data Science, University of California, Los Angeles (09/2021 – 05/2023)" had its bottom half on page 1 and top half on page 2 — html2pdf was cutting horizontally through the glyphs. Cause: earlier PR (PR #23 on 2026-05-13) removed `page-break-inside: avoid` from `<p>` to fix a blank-page-1 bug. That fix was correct AT THE TIME because pagebreak mode included `'avoid-all'`, which cascaded the rule onto wrappers. With `'avoid-all'` now gone, `<p>`-only avoid is safe — added back. Also on `<h3>` so wrapped job titles don't split.

## Important changes

- **The PRESERVE COMPLETENESS rule is fragile.** Llama 3.3 might still over-summarise on edge-case inputs. Worth monitoring user reports.
- **PDF page-break rules now have a known good config.** Documented in ARCHITECTURE.md's "Important constraints" section so future changes don't regress.
