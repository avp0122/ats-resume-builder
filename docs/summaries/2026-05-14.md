# Session summary — 2026-05-14

A bad day. User uploaded the same Cake Resume PDF that PR #16 supposedly fixed in dev, and got the "We could not extract text" error on Vercel. The PDF parses cleanly in local dev — this is a serverless bundling issue, not a parser-capability issue. Spent the day chasing it through five distinct fix attempts before landing on a library swap.

## Completed

### Five fix attempts (PR #26 → #29, merged as #28 + #29)

**Attempt 1 — Multi-path `await import()` chain.** Tried four pdfjs-dist entry points (`legacy/build/pdf.mjs`, `.js`, bare, root) so if one failed bundling, another might work. Combined with `serverComponentsExternalPackages` in `next.config.js` so Next wouldn't try to bundle pdfjs at all. **Result:** Vercel build hung at "Creating an optimized production build" forever — Webpack tried to statically resolve every candidate import and got stuck in a loop. Plus the `@ts-expect-error` directive risked failing the build with "unused directive" if TS happened to resolve it.

**Attempt 2 — `eval('require')` to defeat the bundler.** Used Node's real require at runtime so Webpack couldn't see through to the import. Build succeeded. **Result:** Runtime error "Cannot find module 'pdfjs-dist'" — Vercel's file tracer (nft) only follows literal `require()` calls, not `require(variable)`. With eval-require, nft thought pdfjs was unused and stripped it from `/var/task/node_modules` entirely.

**Attempt 3 — Literal `require()` + `outputFileTracingIncludes`.** Switched to bare `require('pdfjs-dist/legacy/build/pdf.js')` (CJS path so nft can trace) inside the loader function. Added `outputFileTracingIncludes` to `next.config.js` as belt-and-braces. **Result:** Same "Cannot find module" error. nft was still excluding the package; the `outputFileTracingIncludes` config didn't override.

**Attempt 4 — Bundle pdfjs into the lambda (canvas: false alias).** Stopped trying to keep pdfjs external. Removed `serverComponentsExternalPackages`. Added webpack `resolve.alias.canvas = false` (standard pdfjs+webpack workaround). **Result:** Build succeeded, runtime failed with **"Setting up fake worker failed: Cannot find module '/var/task/.next/server/chunks/pdf.worker.mjs'"**. pdfjs 4.x always tries to dynamic-import its worker file, even with `disableWorker: true`. Webpack bundled `pdf.mjs` but didn't emit `pdf.worker.mjs` alongside.

**Attempt 5 — Library swap to `pdf-parse`** (the one that shipped). `pdf-parse` wraps `pdfjs-dist` 1.10.x internally — a much older version that predates the worker requirement. Boring, battle-tested for serverless Node, no extra config. Used the internal entry `pdf-parse/lib/pdf-parse.js` directly to bypass the package's debug code that reads a test PDF from disk at import time (would have failed on serverless). **Result:** Works. Cake Resume extracts cleanly.

### Other changes
- **Always log extractor failures** (PR #29). Pre-fix the `console.warn` was gated on `NODE_ENV !== 'production'`, so Vercel had zero diagnostics when extraction failed. Both attempts (pdf-parse + pdfreader fallback) now log char counts + error messages unconditionally.

## Important changes

- **`pdfjs-dist` is still in `package.json` for now**, just no longer in the active extraction path. Can be removed in a future cleanup. `pdf-parse` brings its own (1.10.x) pdfjs as a sub-dependency.
- **The original `pdfjs-dist` 4.x worker issue is a known pattern.** Any future need to use modern pdfjs in a Next.js serverless context will hit the same problem. Document this in ARCHITECTURE.md.

## Open problems (at end of day)
