# Session summary — 2026-05-13

Heaviest day of feature work. Six PRs landed back-to-back. Multi-tier pricing, support widget overhaul, blurred-preview UX, signup metadata capture, anonymous-upload claim-on-signup, and the first round of PDF page-break tuning.

## Completed

### Shipped to production
- **`profiles` schema prune** (PR #19, merged). Resume-derived fields (`full_name`, `phone`, `location`, `social_links`, `date_of_birth`, `contact_email`) moved off `profiles` — those describe a specific upload, not the account. Migration `008_profiles_user_info_only.sql`. Generate route stops backfilling profile from each upload. Each upload's contact block lives in `resume_uploads`.
- **Support widget: floating popup, not a page** (PR #19). New `components/SupportWidget.tsx` mounted in `app/layout.tsx`. Bottom-right floating button → modal with Title / Email / Phone / Message. Phone optional; later (PR #21) email made required. Migration `009_support_tickets_phone.sql` adds the phone column. Old `/support` page + standalone form removed.
- **Multi-tier Pro pricing** (PR #19). `PRO_TIERS` in `lib/pricing.ts`: 1mo $4.99, 3mo $11.98 (−20%), 1yr $41.92 (−30%). `PlanId` stays `'free' | 'pro'` in the DB; period selection only controls `pro_until` extension. `PricingCard` shows the period selector inside the Pro card.
- **TRC-20 + ERC-20 (replacing BEP-20)** (PR #21). Yesterday's BEP-20 was the wrong call. Reverted to TRC-20 + ERC-20. `lib/crypto.ts` dispatches via `verifyUsdtTransfer(chain, ...)`: TRC-20 → TronGrid REST + base58 addresses + 19 confirmations; ERC-20 → Etherscan V2 (`chainid=1`) + EVM addresses + 12 confirmations. Per-chain validators + normalizer for hash format.
- **Anonymous users: blurred preview + locked download** (PR #19). Was an all-or-nothing PaywallCard; now the preview renders but blurred behind a `BlurGate` overlay with sign-up CTA. `DownloadButton` has a locked state ("Download (sign up)"). Pointer events blocked on the blurred layer so inner Copy button isn't clickable through.
- **Checkout page: no flicker on period/chain toggle** (PR #21). The `useEffect` was clearing `invoice` to null on every change → "Creating invoice…" skeleton flashed. Now keeps the existing card mounted while re-fetching; tiny spinner near Order label signals the refresh.
- **Signup-time client metadata** (PR #20). New columns on `profiles`: `email`, `signup_os`, `signup_browser`, `signup_browser_version`, `signup_country`, `signup_city`, `signup_ip`. `lib/userAgent.ts` parses UA server-side; `lib/geoip.ts` does best-effort lookup via `ipapi.co` (2s timeout, never blocks signup). Migration `010_profiles_signup_meta.sql`.
- **Anonymous resume_uploads + claim-on-signup** (PR #22). `resume_uploads.user_id` was `NOT NULL`. Migration `011_resume_uploads_anon.sql` makes it nullable, adds `anon_id`, enforces XOR constraint. New cookie `kairesume_anon_id` (signed, 1-year). On signup: `UPDATE resume_uploads SET user_id = $new, anon_id = NULL WHERE anon_id = $cookie`, then clears the cookie.
- **Route progress bar visibly reaches 100%** (PR #19's chain). Pre-fix the 250ms hide timer minus the 180ms width transition gave the bar only ~70ms at 100% — eye doesn't register, looks stuck at 80–90%. Now uses `requestAnimationFrame` + two-stage hide so it visibly fills before fading.
- **bg-clip-text descender clip fix** (PR #19's chain). "Simple, transparent pricing" was getting the "p" tails clipped. Fix in `.gradient-text` CSS: `line-height: 1.15` + `padding-bottom: 0.1em`.
- **PDF first-page-blank fix** (PR #23). `html2pdf` `pagebreak.mode: ['avoid-all', 'css', 'legacy']` was injecting `page-break-inside: avoid` on every wrapper, pushing the entire resume body to page 2. Removed `'avoid-all'`. Same idea at the template level — removed `page-break-inside: avoid` from `<p>`. (Later regretted on 2026-05-17 — caused mid-glyph slicing. Put back, but only on `<p>` directly.)
- **AEO/GEO/SEO surface area** (PR #23). `/llms.txt` (Markdown TOC for AI answer engines per llmstxt.org with sentence-level Q/A). Three JSON-LD blobs: `Organization`, `WebApplication` (with all Pro tier `Offer`s), `HowTo`. Expanded FAQ from 5 → 10 entries. Per-page metadata for `/pricing`, `/privacy`, `/terms`.
- **Drag & drop file upload** (PR #25). Drop zone now accepts drag-and-drop alongside click-to-browse. `dragDepth` counter so nested children don't flicker the highlight. New `.file-drop.is-dragging` CSS. Single `acceptResumeFile()` helper for both click + drop paths.
- **TPM budget round 3 + 413 auto-retry** (PR #25). Real-world breach "Requested 8524, Limit 8000" even after previous rounds. `estimateTokens` divisor `3.5` → `3`; `SAFETY_MARGIN` `800` → `1200`; per-input caps JD `1500`, resume `2000`. **413 auto-recovery**: `lib/llm.ts` catches 413 and retries with shrunk inputs (JD → 40%, resume → 60%).

## Important changes

- **Quota state is now refreshed on every `/api/generate` response.** The `RemainingBanner` / `UpgradePromo` pair was driven by `quota` state that only loaded once on mount. After a generation completed, the banner stayed stale until the user reloaded. Fixed via `applyQuotaFromResponse()` helper that maps both success and 402 responses into `QuotaInfo`.
- **The "stale banner vs error" contradiction is gone.** A signed-in free user who just hit the cap used to see both "2 of 3 remaining" mid-page AND "You've used all 3 generations" error under the form. The banner now flips to `UpgradePromo` the moment `remaining === 0`.
